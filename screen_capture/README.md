# 屏幕捕获与鼠标坐标/颜色监控工具

一个基于 Tkinter + PyAutoGUI + MSS + Pillow 的桌面辅助工具，提供以下核心能力：

- 实时显示鼠标屏幕坐标与像素颜色（RGB / HEX）
- 支持相对坐标模式（可自定义基准点，F8 设置）
- 记录最近的坐标历史并可快速复制
- 可选择输出多种坐标格式（Python / JavaScript / CSS）
- 提供一个可拖动、可缩放、自定义大小的屏幕区域捕获窗口
- 第二个显示窗口实时预览所选区域内容，可叠加网格与标尺
- 支持截图单帧保存（右键）与定时录制（定时保存 PNG 序列）
- 支持预设分辨率快速调整（如 1280×720 / 1920×1080 / 4K 等）
- 支持锁定宽高比调整大小
- 支持滚轮缩放预览内容

## 10. OCR 文字识别 (多后端 & 新增特性)
### 功能简介
在预览工具栏新增“识别文字”按钮（或快捷键 Ctrl+O），支持多后端自动选择：

1. 优先尝试 PaddleOCR（已安装 & 可初始化）
2. 若 PaddleOCR 缺失或初始化失败自动回退 RapidOCR（`rapidocr-onnxruntime`）

识别结果弹窗中可以：
1. 查看识别文本（自动按行拼接）
2. 编辑文本内容
3. 复制到剪贴板
4. 另存为 TXT
5. 重新识别（例如切换处理器或来源后）
 6. 过滤空行 / 查看统计（行数 & 字符数）

### 新增特性 (A / B / C)
| 标记 | 功能 | 描述 |
|------|------|------|
| A | 本地轻量模型自动回退 | 若使用本地 `ch_ppocr_mobile_v2.0_*` 目录加载的轻量 Paddle 模型并得到 0 行结果，会弹出提示，确认后自动切换到官方远程 PP-OCRv4 模型并立即重新识别（只提示一次）。 |
| B | 文本后处理 | 识别结果行将自动执行：折叠多余空白、去除中文字符之间无意义空格、清理标点前后空格、删除完全空行。 |
| C | 模型信息显示 | 结果信息条新增 “模型: xxx” 字段，区分 `paddle-local-mobilev2.0` / `paddle-remote` / `rapidocr`。 |

信息条示例：
```
来源: 处理后 | 行数: 12 | 总耗时: 0.84s | 预处理: 0.08s (缩放:0.03s 基础:0.05s) | 推理: 0.63s | 后处理: 0.03s | 输入: 1280x640 | 语言: ch | 后端: paddle | 模型: paddle-remote
```

### 模型来源说明
- paddle-local-mobilev2.0：使用你手动提供的本地轻量 v2.0 推理目录（速度快、可能召回率低）。
- paddle-remote：未提供本地目录，PaddleOCR 自动下载 PP-OCRv4（精度较高）。
- rapidocr：Paddle 初始化失败或未安装时 fallback 到 RapidOCR。 

可通过设置 `self.use_local_ocr_models = True/False`（或未来添加的设置 UI）来控制是否尝试加载本地目录。

### 来源选择
在“处理...”图像处理面板底部新增 OCR 来源单选：
- 原始图像：不经过处理器
- 处理后图像：按当前勾选顺序执行所有启用的处理器（如灰度、反相、边缘等），然后送入 OCR

选择“处理后图像”可以借助前置滤波或增强来提升某些场景下的识别率，特别是对比度低或背景噪声较多的界面。

### 懒加载与后端回退机制
首次点击“识别文字”时才加载 OCR 引擎：
- 如果 paddleocr + paddlepaddle 可用 → 初始化并标记后端 `paddle`
- 否则尝试导入 rapidocr-onnxruntime → 标记后端 `rapidocr`
- 两者都失败 → 弹窗提示安装命令

识别结果信息行会显示：`后端: paddle` 或 `后端: rapidocr` 便于确认来源。

### 依赖安装
按需安装（体积较大时可只装其一）：

PaddleOCR（功能更全面，检测 + 方向分类 + 多语言）：
```
pip install paddlepaddle -i https://pypi.tuna.tsinghua.edu.cn/simple
pip install paddleocr
```

RapidOCR（轻量 + ONNXRuntime，仅需一个包即可快速运行）：
```
pip install rapidocr-onnxruntime
```

建议：
- 需要更高的多语言、方向分类或后续可视化 → 安装 Paddle
- 只需轻量快速文本行识别 → 可仅安装 RapidOCR

（程序会自动从已安装的可用后端中选择）

### 数据结构（内部缓存）
程序内部将最后一次识别结构保存为：
```
{
	"lines": [
		{ "box": [[x1,y1],...], "text": "识别结果", "score": 0.98 },
		...
	],
	"source": "raw" | "processed",
	"timestamp": 1690000000.0,
	"duration": 0.73
	"backend": "paddle" | "rapidocr",
	"model": "paddle-remote" | "paddle-local-mobilev2.0" | "rapidocr"
}
```
为未来显示检测框（方案 C）与框交互预留。

### 使用建议
1. 若识别结果为空或乱码，可尝试勾选“灰度”再用“处理后图像”识别。
2. 对高对比 UI（黑底白字）可先反相再识别（实验性，视具体情况）。
3. 识别窗口可保持打开，重复点击“重新识别”方便比对不同处理组合。
4. 本地轻量模型识别为空时接受回退提示可显著提升召回。
5. 关注信息条中的 “缩放” 与 “输入尺寸” 判断是否因为尺寸过小导致召回低。

### 后续可扩展（方案 C 预留）
- 在预览窗口叠加文字检测框（可开关）
- 悬停/点击高亮对应行并在结果窗口同步定位
- 置信度低于阈值的行单独标色或折叠

| 按钮 | 说明 |

额外：结果窗口包含 2 个标签页：
1. 文本：可编辑的识别结果；按钮操作主要作用此区域。
2. 信息：显示本次识别的元数据：
   - 后端与模型：`paddle` / `rapidocr` 及模型来源（如 `paddle-remote` / `paddle-local-mobilev2.0` / `rapidocr`）
   - 行数：解析与后处理后保留的文本行数
   - 输入尺寸：送入 OCR 引擎的图像分辨率（可能因最大边约束而缩放）
   - 耗时统计：总耗时与推理阶段、后处理阶段拆分（秒，三位小数）

刷新策略：
- 首次打开窗口调用一次内部 `_refresh_ocr_meta()`。
- 每次成功识别后内部更新 `self.last_ocr_result`，并可（已在实现中）调用 `_refresh_ocr_meta()` 立即刷新信息页，同时刷新缩略区域。

(新增) 置信度过滤：OCR 结果窗口顶部第二行提供滑块“置信度≥”，拖动后立即只显示分数 >= 阈值的行；信息页行数显示为 过滤后/总行数；阈值自动写入用户配置并在下次启动后恢复。

## 14. AI 能力快速指南 (更多详见项目根目录 `readme_ai.md`)

| 模块 | 功能 | 入口 |
|------|------|------|
| AI 对话 | 多轮对话 / 插入 OCR 文本 / 模板提示 / System Prompt | 右侧 AI 面板 “AI对话” 按钮或面板按钮 |
| 图像多模态分析 | 将当前截取图像发送给多模态模型并获取语义描述/分析 | AI 对话窗口 → 顶部“图像分析”菜单（分析当前画面/选择图片/粘贴板） |
| 课堂行为分析（JSON 输出） | 输出严格 JSON（结构化结果，便于系统对接/可视化） | AI 对话窗口 → 图像分析 → 课堂行为分析（JSON 输出）… |
| OCR 文本 AI 编排 | 对识别文本执行 纠错/润色/摘要 组合处理 | OCR 面板 “AI编排” 按钮 |
| 事件总线 | 触发 `ocr_finished` 与 `ai_reply` 供状态栏/其它扩展订阅 | 内部 `event_bus` |

快速流程：捕获区域 → OCR → （可选：AI编排）→ 打开 AI 对话窗口 → 图像分析（含课堂行为分析/JSON 输出）。

API Key：环境变量 > keyring > 运行时输入；不会写入仓库。

更多细节、FAQ 与 Roadmap：参见根目录 `readme_ai.md`。

## 16. 课堂行为分析与可视化（新增）

本功能帮助你基于课堂图片快速得到标准化分析结果，并可一键生成摘要 PNG 图片。

1) 在 UI 中使用
- 打开 AI 对话窗口 → 图像分析 →
	- 课堂行为分析…（人类可读的 Markdown 报告，支持自定义 Prompt）
	- 课堂行为分析（JSON 输出）…（强制严格 JSON，仅输出结构化结果）
- 选择图片；若同目录存在同名 `*_counts.json` 会自动附带（可手选或跳过）
- 结果弹窗底部按钮：
	- 复制内容、另存为…
	- 另存为 JSON → 生成可视化图（一键保存 JSON 并生成同名 `_summary.png` 图）

说明：结果不是纯 JSON 时，程序会自动尝试从文本中提取 JSON 片段（支持 ```json 代码块 或 最大花括号块），尽量完成可视化。

2) 命令行可视化（可选）
- 将结构化结果保存为 JSON 后，可用以下命令生成 PNG 摘要图：
	```powershell
	python .\student_behavior_ai\visualize.py --json .\report.json --out .\report_summary.png --title "课堂行为分析"
	```
- 渲染内容包含：摘要、关键要点、风险与建议、指标条（低头率/看手机/阅读记笔记/举手/分心）、评分条（专注度/活跃度）、局限性等。

3) Prompt 模板与约定
- 推荐使用 `docs/behavior_prompts.md` 中的模板：
	- Markdown 报告模板（面向老师汇报）
	- 严格 JSON 模板（系统对接/可视化）
- JSON 字段包含：summary、observations[]、metrics{head_down_rate, phone_usage_rate, reading_note_rate, hand_raise_rate, distracted_rate, interaction_level, focus_score, activity_score}、risks[]、suggestions[]、limitations[]、confidence。

4) 依赖与兼容
- 可视化基于 Pillow；已对不同 Pillow 版本做了兼容（文本测量/圆角矩形回退）。
- 字体：Windows 优先使用微软雅黑/黑体，缺失时会回退到默认字体。
