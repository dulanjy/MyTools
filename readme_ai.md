| 功能 | 描述 | 触发位置 |
|------|------|----------|
| 截取区域实时预览 | 动态显示被选定区域图像，可放大/网格/标尺辅助 | 主窗口左侧图像区 |

| 事件总线 | 解耦 OCR 完成 / AI 回复 与 UI 刷新逻辑 | 内部 (event_bus) |

## 2. 安装与依赖
核心依赖（根据实际 requirements 精简列出）：
```
Pillow
paddleocr
keyring
requests
zai  # Zhipu AI Python SDK（用于 glm-4/4.5v 等）
```
可选依赖：
- `pyperclip`：提供系统剪贴板复制（有则用，无则回退 Tk clipboard）
- `numpy`：部分图像处理器（如边缘增强）

安装示例：
```bash
pip install -r requirements.txt
# 或最小 AI 子集
pip install paddleocr Pillow keyring requests
```

> 若使用本地 PaddleOCR 模型，请确保 `inference/` 下对应模型解压到位。

## 3. API Key 配置与安全
AI 模型访问使用统一 `ai_client`：
1. **加载顺序**：环境变量 → keyring（系统安全存储） → 运行时提示（后续写入 keyring）。
2. 设置示例（临时会话）：
   - Windows PowerShell: `setx ZHIPU_API_KEY "your_key"` (永久写入用户环境)
   - 或运行程序后按提示输入。
3. Key 不会被写入仓库文件；推荐使用系统凭据管理器。
4. 变更 Key：清理系统凭据中对应项或重新输入覆盖。

## 4. 界面结构与术语
```
+-----------------------------------------------------------+
| 左侧：图标侧栏 (导航)  | 中间：主内容（预览/OCR/处理/录制/设置） |
+------------------------+-----------------------------------+
| 右侧：折叠快捷卡片区   |                                   |
+-----------------------------------------------------------+
| 底部状态栏: OCR | AI | Info ...                           |
+-----------------------------------------------------------+
```
- AI 对话：改为“独立窗口（Toplevel）”，不再占用右侧卡片或主布局空间。
   - 顶部控件：系统指令、图像分析(菜单)、置顶(勾选)、取消、发送/发送OCR、状态显示
   - 中部：历史对话区（带角色着色）
   - 模板区：常用提示一键插入（可折叠/显示）
   - 底部：输入区 + 上下文统计（消息/字符数）
- 右侧快捷卡片中“AI 助手(快捷)”仅保留“打开 AI 对话”按钮。
- 事件总线：`ocr_finished` / `ai_reply` 驱动状态栏等联动。

## 5. 使用流程
### 5.1 截屏与 OCR
1. 启动程序后拖拽/调整截取区域。
2. 点击 “OCR” 按钮执行识别。若启用图像处理（灰度、反相等）会先处理后识别。
3. 结果出现在中部 OCR 面板 “结果” 页签；可：复制 / 另存为 / 滤空行 / 统计 / 设置阈值。
4. “元信息” 页签显示：后端、模型、行数、输入尺寸、耗时、推理阶段耗时等。

补充（文件/文件夹识别）：
- 在 OCR 面板顶部按钮栏，可点击“从文件识别…”选择一张图片直接识别；左侧预览将显示所选图作为“原”图，同时后台执行 OCR 并在“结果”页签呈现文本。
- 可点击“从文件夹识别…”对指定目录下的图片批量识别（当前串行执行，完成数量会在状态栏显示）。
- 图像区域右键菜单已精简为仅“保存当前截图”，不再提供“从文件识别…”，避免入口分散。

### 5.2 AI 对话面板
功能点：
- 支持多轮上下文（暂无限制策略，未来支持裁剪说明）。
- “插入 OCR 文本” 会把当前识别文本作为 user 消息追加。
- “System Prompt” 设置角色（例如："你是专业中英翻译"）。
- 快速模板：展开后点击按钮自动拼接提示到输入框。
- 取消：生成中可中断（依赖 AsyncCall 线程标志）。
- 上下文统计：显示 tokens/字符估算（未来可裁剪）。

### 5.3 图像多模态分析（AI 窗口菜单 + Prompt）
入口：打开 AI 对话窗口 → 顶部“图像分析”下拉菜单，包含：
- 分析当前画面：使用当前捕获画面（`last_pil`）
- 选择图片分析…：弹出文件选择器，选中后分析
- 粘贴板图片分析：直接读取剪贴板中的图片或图片文件路径

流程与特性：
1) 先弹出“自定义 Prompt”对话框（可记住上次输入）。
2) 分析弹窗会显示元信息（原始/缩放后尺寸、缩放比例、格式、最终字节大小）。
3) 支持“取消”按钮联动（UI 层取消）：无法中途打断请求，但返回后呈现“已取消”。
4) 超时/稳定性：
   - ai_client.analyze_image 默认超时 30s（SDK 支持则生效），并在 UI 层有 35s 看门狗超时兜底。
   - 超大图会在客户端按最长边 1600px 等比缩放，并按 PNG/JPEG 选择保存（JPEG 时 quality/optimize 生效）。
5) 失败时：弹窗底部提供“复制错误信息”按钮，便于反馈与排查；日志会输出详细元信息与异常堆栈。

### 5.4 OCR 文本 AI 编排
1. 在 OCR 面板中点击 “AI编排”。
2. 选择需要操作：纠错 / 润色 / 摘要，可附加额外指令。
3. 执行后生成结果写入 Notebook 新/复用的 “AI编排” 页签。
4. 长文本会截断（默认前 8000 字），在提示首部加截断说明。

## 6. 事件总线 (Event Bus)
文件：`screen_capture/event_bus.py`
接口：
```python
bus = get_global_bus()
# 订阅
bus.on('ocr_finished', lambda payload: ...)
# 一次性
bus.once('ai_reply', handler)
# 取消订阅
bus.off('ocr_finished', handler)
# 触发
bus.emit('ocr_finished', data)
```
当前触发点：
- OCR 完成：`emit('ocr_finished', {...meta...})`
- AI 回复：`emit('ai_reply', {'role': 'assistant', 'content': ...})`
用途：
- 解耦状态栏刷新
- 未来：日志记录、调试面板、外部插件扩展

| 使用处理后图像 | OCR 来源（原图/处理后） | 工具栏短标签 | 已在状态中 |
| 置信度阈值 | 滤低质量行 | OCR 面板滑块 | 计划支持 |
| OCR 语言 | PaddleOCR 语言配置 | 菜单“切换语言” | 计划支持 |
| System Prompt | AI 初始角色指令 | AI 窗口顶部“系统指令” | 计划支持 |

## 8. 常见问题 FAQ
**Q1: 为什么“选择图片分析…”点了没反应，且窗口无法关闭？**  
A: 这是菜单 grab 与对话框的焦点冲突问题，已通过在菜单回调中使用 `after(10, ...)` 延迟执行修复；若仍偶发，可把延迟增至 50~100ms。

**Q2: 图像分析/聊天会“卡住”很久？**  
A: 已加入请求超时（默认 30s）与 UI 看门狗（35s）；同时会对大图进行等比缩放与压缩，显著减少等待时间。若依旧缓慢，请检查网络/代理；必要时缩小图片或简化 Prompt。

**Q3: 剪贴板分析提示“未检测到图片”？**  
A: 请确认剪贴板中的是图片或图片文件路径列表（Windows 资源管理器复制一张图片文件也可）。

**Q4: OCR 面板不见了？**  
A: 通过导航或再次执行 OCR 即可恢复；或检查右侧折叠卡片中是否隐藏。

**Q5: 想自定义更多图像预处理？**  
A: 在 `_register_builtin_processors` 中添加处理器字典并刷新处理器面板。

## 10. 贡献指引 (AI 部分)
### 代码结构约定
- `capture.py`：主界面与交互逻辑（已采用左侧图标侧栏 + 右侧折叠快捷卡片）。
- `ui/ai_chat.py`：AI 对话（独立窗口）、图像分析菜单与弹窗（Prompt、元信息、超时、取消）。
- `ai_client.py`：模型访问封装（文本 / 图像 多模态；包含超时、缩放/压缩、元信息日志）。
- `event_bus.py`：全局事件分发。
- 处理器扩展：于 `_register_builtin_processors` 中注册；建议保持纯函数 + 小参数集。

### PR 建议
1. 避免在 UI 回调里执行阻塞网络请求；使用 `AsyncCall`。
2. 新事件请在文档中登记（本文件 6 节）。
3. 增加可视化/快捷键前请考虑与现有绑定冲突。
4. 若改动公共 API（如 ai_client.chat 签名）需添加兼容层或更新调用点。

### 日志与调试
- 切换日志级别：Overflow → 日志级别(循环)
- 建议在新特性中使用 `logger.debug` 精准输出输入/耗时。
